Tema 1

1.Introduccion a las aplicaciones web

En términos simples, las aplicaciones web son sitios web que, cuando se combinan con un servidor, proporcionan funcionalidades que nos permiten interactuar con los usuarios en Internet y nos permiten comunicarnos con otras bases de datos.

Estas aplicaciones son accesibles desde cualquier dispositivo con conexión a internet y un navegador compatible, lo que las hace extremadamente versátiles y convenientes. En lugar de depender de software instalado localmente, las aplicaciones web aprovechan la potencia del navegador para proporcionar una experiencia de usuario fluida y dinámica. El desarrollo de aplicaciones web implica una combinación de tecnologías como HTML, CSS y JavaScript para crear la interfaz de usuario y la lógica de la aplicación. Además, hay frameworks y bibliotecas como React, Angular y Vue.js que simplifican y aceleran el proceso de desarrollo.

1.1 Evolucion de las aplicaciones web

Desde que Internet y la Web fueron creados, una cantidad infinita de posibilidades se han abierto, quizás, la más importante y usual sea el acceso de datos e información desde cualquier sitio. Las personas que se dedican al desarrollo de aplicaciones podrían considerar esto como un desafío, ya que los avances tecnológicos de estos últimos años exigen aplicaciones más rápidas, ligeras y robustas que permitan utilizar la Web. Sin lugar a dudas, Internet es una de las últimas tecnologías que más rápidamente se está desarrollando para su introducción en los hogares. Las previsiones apuntan a su integración como un electrodoméstico más, con unas capacidades y servicios que evolucionaran rápidamente. Por ese motivo, cada vez más adquieren importancia en las respuestas de los usuarios frente a la implantación de estas nuevas tecnologías.

1.2 Arquitectura de las aplicaciones web

Las aplicaciones Web están basadas en el modelo Cliente/Servidor que gestionan servidores web, y que utilizan como interfaz páginas web. Las páginas Web son el componente principal de una aplicación o sitio Web. Los browsers piden páginas (almacenadas o creadas dinámicamente) con información a los servidores Web. En algunos ambientes de desarrollo de aplicaciones Web, las páginas contienen código HTML y scripts dinámicos, que son ejecutados por el servidor antes de entregar la página.

1.3 Tecnologias para el desarrollo de aplicaciones web

HTML, CSS y JavaScript: estas tres tecnologías son fundamentales para el desarrollo web. HTML proporciona la estructura de la página, CSS se utiliza para el diseño y estilos visuales, y JavaScript permite la interactividad y la lógica de la página. Frameworks de JavaScript: existen varios frameworks de JavaScript populares, como React, Angular y Vue.js, que facilitan el desarrollo de aplicaciones web complejas. Framework de back-end: existen varios frameworks de back-end populares, como Express.js (para Node.js) y Ruby on Rails, que facilitan el desarrollo de aplicaciones web en el lado del servidor. Bases de datos: MongoDB, MySQL y PostgreSQL son algunas de las bases de datos más populares utilizadas para almacenar y recuperar datos en una aplicación web. Herramientas de desarrollo: existen muchas herramientas de desarrollo disponibles para facilitar el proceso de desarrollo, como Git para el control de versiones, y Webpack para la gestión de dependencias y la optimización del rendimiento.

1.4 Planificacion de aplicaciones web

La planificación de una web es un simple esquema o boceto de cómo y qué es lo que quieres presentar en tu página web, es decir, las páginas que crearás y qué información pondrás en cada una. Ver todas las piezas antes de montarlas siempre es de gran ayuda. La planificación más básica de tu página web puede consistir simplemente en un esquema del menú de navegación. La primera etapa de cualquier proyecto web es el briefing, cuando el cliente expone sus necesidades. Presenta referencias, y cuando es posible, informa el presupuesto disponible. Es fundamental que el briefing sea bien completo y mucho de eso depende del desarrollador, que en general está más familiarizado con la tecnología que con el cliente. Por lo tanto, realiza muchas preguntas, estimula la participación del cliente y alinea las expectativas.

Tema 2

2 HTML, XML Y CSS

- HTML es el lenguaje de marcado principal para crear y estructurar páginas web. Utiliza una serie de elementos y etiquetas para definir la estructura y el contenido de una página web. Por ejemplo, los encabezados, párrafos, enlaces, imágenes y otros elementos multimedia se definen mediante etiquetas HTML.

- XML es un lenguaje de marcado que define reglas para la codificación de documentos en un formato que es legible tanto para humanos como para máquinas. A diferencia de HTML, XML no tiene un conjunto predefinido de etiquetas; en cambio, permite a los usuarios definir sus propias etiquetas para estructurar datos de una manera que sea lógica y significativa para su aplicación específica.

- CSS es un lenguaje de diseño utilizado para describir la presentación de un documento escrito en HTML o XML. CSS se encarga del estilo y la disposición visual de las páginas web, como los colores, las fuentes, los márgenes y la alineación de los elementos. CSS permite separar el contenido de la presentación, facilitando el mantenimiento y mejorando la flexibilidad del diseño.

2.1 Introduccion

En la era digital, la creación y gestión de contenido web son esenciales para la comunicación, el comercio, la educación y el entretenimiento. Tres tecnologías fundamentales que forman la base del desarrollo web son HTML, XML y CSS. Cada una de estas tecnologías tiene un propósito específico y, cuando se combinan, permiten la creación de páginas web dinámicas, estructuradas y visualmente atractivas. HTML es el pilar sobre el cual se construyen las páginas web. Como lenguaje de marcado de hipertexto, HTML proporciona la estructura básica de una página web mediante el uso de etiquetas. XML es una tecnología flexible que se utiliza principalmente para el almacenamiento y transporte de datos. A diferencia de HTML, XML no tiene un conjunto predefinido de etiquetas; en su lugar, permite a los desarrolladores crear etiquetas personalizadas que describen de manera precisa los datos que contienen. CSS es el lenguaje de estilos utilizado para controlar la presentación visual de las páginas web. Con CSS, los desarrolladores pueden aplicar estilos a los elementos HTML, como colores, fuentes, márgenes, y layout.

2.2 Estructura global de un documento web

La estructura global de un documento web es esencial para organizar y presentar contenido de manera clara y accesible. Un documento web bien estructurado facilita la lectura por parte de los navegadores y mejora la experiencia del usuario. En HTML, la estructura global se compone de varias secciones fundamentales que deben seguir un orden lógico. Siguiendo esta estructura, los desarrolladores pueden asegurarse de que sus páginas sean accesibles, mantenibles y atractivas para los usuarios. Esta estructura también facilita la correcta interpretación y renderización del contenido por parte de los navegadores web.

2.3 Elementos basicos: texto, vinculos, listas, tablas, objetos, imagenes y aplicaciones

En resumen, los elementos básicos en HTML son fundamentales para la construcción de páginas web. Aquí hay un resumen de los elementos que hemos discutido:

- Texto: Se utiliza la etiqueta p> para crear párrafos y span> para texto en línea.

- Vínculos: Se crean con la etiqueta a> y se utilizan para enlazar a otras páginas web o recursos.

- Listas: Se pueden crear listas desordenadas con ul> y li>, o listas ordenadas con ol> y li>.

- Tablas: Se definen con table>, tr> para filas y td> para celdas, permitiendo organizar datos en forma de tabla.

- Objetos: Se pueden incrustar usando la etiqueta object> para agregar elementos multimedia o aplicaciones.

- Imágenes: Se muestran con la etiqueta img> y se enlazan a la ubicación del archivo de imagen mediante el atributo src.

- Aplicaciones: Se pueden incrustar utilizando la etiqueta iframe>, lo que permite mostrar contenido interactivo de otras fuentes dentro de una página web.

2.4 Formularios

Los formularios en HTML son una herramienta fundamental para la recolección de datos de los usuarios. Permiten a los visitantes de un sitio web enviar información que puede ser procesada por el servidor. Los formularios están compuestos por diversos elementos de entrada, como cuadros de texto, botones de radio, casillas de verificación, menús desplegables y botones de envío. Estructura Básica de un Formulario Un formulario en HTML se define utilizando la etiqueta form. Los atributos más comunes de esta etiqueta son action, que especifica la URL a la que se enviará el formulario, y method, que define el método HTTP (GET o POST) que se utilizará para enviar los datos. La etiqueta FORM actúa como una especie de contenedor para almacenar elementos que permiten al usuario seleccionar o introducir datos. Todos los datos se enviarán a la dirección URL indicada en el atributo ACTION de la etiqueta FORM, por el método indicado en el atributo METHOD. Se puede insertar cualquier elemento HTML en una etiqueta FORM (como texto, botones, tablas y enlaces), pero los elementos interactivos son los más interesantes.

2.5 Lenguajes de presentacion en documentos web

Los lenguajes de presentación en documentos web son fundamentales para definir el estilo y la apariencia visual de las páginas web. Los dos lenguajes principales son CSS (Cascading Style Sheets) y XSL (eXtensible Stylesheet Language). Aquí te explico cada uno: CSS es el lenguaje estándar para la presentación de documentos web. Permite definir estilos para elementos HTML, como colores, fuentes, márgenes, tamaños, posicionamiento y otros atributos visuales. CSS se utiliza para separar la estructura de un documento HTML de su presentación, lo que facilita la creación de sitios web consistentes y con un diseño atractivo. XSL es un lenguaje de marcado utilizado para transformar y presentar documentos XML. Dentro de XSL, existe un subconjunto llamado XSL-FO (Formatting Objects), que se utiliza específicamente para definir la presentación visual de documentos XML. Aunque no es tan comúnmente utilizado como CSS para el diseño web, XSL es poderoso para la transformación de datos XML y la generación de informes estructurados.

2.6 Selectores

Los selectores son mecanismos que se utilizan para localizar e interactuar con elementos en una aplicación de interfaz gráfica de usuario (GUI). Estos elementos a los que a menudo se hace referencia como controles pueden ser cualquier cosa, desde ventanas a botones o cajas de combinación. Los selectores utilizan los atributos internos del elemento para identificar el elemento. En una página web, los elementos son las etiquetas HTML que componen la página web y los atributos internos del elemento son los atributos HTML definidos en el elemento, como las clases, los ID y el nombre de etiqueta HTML.

2.7 Modelo de cajas

El modelo de cajas o “box model” es seguramente la característica más importante del lenguaje de hojas de estilos CSS, ya que condiciona el diseño de todas las páginas web. El modelo de cajas es el comportamiento de CSS que hace que todos los elementos de las páginas se representen mediante cajas rectangulares. Las cajas de una página se crean automáticamente. Cada vez que se inserta una etiqueta HTML, se crea una nueva caja rectangular que encierra los contenidos de ese elemento. Los navegadores crean y colocan las cajas de forma automática, pero CSS permite modificar todas sus características. Cada una de las cajas está formada por seis partes

Tema 5

5 Computo en la nube y servicios

El cómputo en la nube, también conocido como cloud computing, se refiere a la entrega de servicios de computación, como servidores, almacenamiento, bases de datos, redes, software, análisis e inteligencia, a través de Internet ("la nube") para ofrecer una flexibilidad, escalabilidad y eficiencia mejoradas en comparación con las soluciones tradicionales de infraestructura de TI local.

El cómputo en la nube ha transformado la forma en que las organizaciones gestionan y utilizan la tecnología de la información, ofreciendo una gama de servicios y beneficios que impulsan la innovación, la eficiencia y la competitividad en el mercado. Sin embargo, también plantea desafíos únicos que deben abordarse para garantizar su adopción y uso efectivos.

5.1 Conceptos generales

Nube Pública: Los servicios de nube son ofrecidos por proveedores de servicios en una infraestructura compartida y accesibles a través de Internet público. Ejemplos incluyen AWS (Amazon Web Services), Microsoft Azure, Google Cloud Platform.

Nube Privada: La infraestructura de la nube está dedicada exclusivamente a una sola organización y puede ser gestionada internamente o por un tercero. Ofrece mayor control y seguridad, pero también requiere una inversión y mantenimiento significativos.

Nube Híbrida: Combina recursos de nube pública y privada, permitiendo la portabilidad de datos y aplicaciones entre ambos entornos. Proporciona flexibilidad y escalabilidad, así como control sobre datos sensibles y críticos.

5.2 Tipos de servicio en la nube

Los tres tipos principales de computación en la nube son la nube pública, la privada y la híbrida. Dentro de estos modelos de implementación, hay cuatro servicios principales: infraestructura como servicio (IaaS), plataforma como servicio (PaaS), software como servicio (SaaS) y computación sin servidores.

El tipo de modelo de implementación en la nube y modelo de servicio en la nube que elijas variará según las inversiones de TI existentes, los requisitos empresariales y los resultados que esperas lograr.

5.5 Plataformas tecnologicas

Una plataforma tecnológica es la base para crear y ejecutar aplicaciones de negocio. La plataforma permite a los usuarios ejecutar sus aplicaciones de sin problemas sin tener que preocuparse por la tecnología que les da soporte. Al mismo tiempo, permite al personal técnico ampliar, mejorar o actualizar rápidamente el software de aplicación, y aumentar la velocidad del negocio. En general, una plataforma tecnológica moderna incluye analíticas, bases de datos y gestión de datos, herramientas para el desarrollo y ampliación de aplicaciones, integración y tecnologías inteligentes, tales como inteligencia artificial (IA), machine learning, e internet de las cosas (IoT).

5.6 Seguridad e interoperabilidad

La seguridad e interoperabilidad son aspectos críticos en el desarrollo de aplicaciones web, especialmente en contextos donde se manejan datos sensibles o se integran diferentes sistemas. La seguridad en aplicaciones web abarca una serie de prácticas y tecnologías diseñadas para proteger los datos y la funcionalidad de las aplicaciones web contra amenazas y vulnerabilidades.

La interoperabilidad se refiere a la capacidad de diferentes sistemas y aplicaciones para trabajar juntos, compartir datos y servicios sin problemas. Esto es crucial en entornos donde múltiples sistemas deben integrarse y comunicarse eficazmente.
